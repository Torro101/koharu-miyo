What MIYO is

MIYO is an Android comic reader built for large libraries, online sources, and reliable offline reading. It focuses on a clean Material interface, strong source/plugin boundaries, stable downloads, and practical performance on low-end and modern devices.

What MIYO is not

MIYO is not a manga host, content mirror, or source catalog website. Users control their own local files and source configuration, and they are responsible for the services they access.

Application layers

  1. UIMaterial screens for history, favorites, explore, feed, local storage, reader, downloads, and settings.
  2. ReaderChapter pages, cache, page state, reading progress, and offline fallback behavior.
  3. DownloadsWorkers, notifications, local directories, cancellation, validation, and storage cleanup.
  4. SourcesParser logic, source settings, authentication, captcha handoff, Cloudflare/browser flows, and plugin boundaries.
  5. NativeC++ helpers for image probing, archive work, and performance-sensitive refinement paths.
  6. ReleaseGitHub Actions builds, release APK artifacts, model packages, and in-app update metadata.

Compatible plugins

The current external plugin path is for Kotatsu-compatible JAR parser plugins. GitHub release imports search for JAR assets and load them through the app's parser bridge.

  • Use JAR plugins built for the MIYO/Kotatsu parser API.
  • Update both MIYO and the plugin when compatibility errors occur.

Keiyoushi and Mihon APKs

Keiyoushi repositories are detected so users receive a clear message, but those extensions are Mihon/Tachiyomi APKs. They require a dedicated compatibility runtime before MIYO can run them safely.

GitHub Pages deployment

This website is a static GitHub Pages site. It can publish from the `main` branch root without a build step.

  • Source: Deploy from a branch.

Custom domain

The expected default URL is `https://torro101.github.io/koharu-miyo/`. A domain such as `koharu.miyo` should only be added after the domain is actually owned and DNS points to GitHub Pages. This site intentionally does not include a CNAME yet.
