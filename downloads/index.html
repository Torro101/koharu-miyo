Android update compatibility

Android only installs a new APK over an existing installation when the package name and signing certificate match. MIYO release builds use the package ID:

org.koharu.miyo

  • Keep the same signing key for all public releases.
  • Use Gradle-defined app versions, not APK metadata patched after build.
  • Use GitHub Releases as the update source.

Install source

Download APKs only from the official repository release page unless you intentionally build the app yourself from source.

Refinement model distribution

The main APK should stay small. The general bundled model is part of the release APK when configured by CI; optional models are release assets that users can download when needed.

Why not mirror APKs here?

Keeping APKs in GitHub Releases avoids duplicate outdated files in the website repo and keeps checksums, release notes, and update history in one place.
