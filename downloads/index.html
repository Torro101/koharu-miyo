Android update rules

Android only installs an update over an existing app when the application ID and signing certificate match.

  • Package: org.koharu.miyo.
  • Future releases must keep the same signing key.
  • Version metadata should be changed in Gradle source, not patched after build.

Safe install checklist

Install from the GitHub Release page and avoid random reuploads. If Android reports a signature mismatch, uninstalling first turns the update into a clean install and may remove local app data.

Models

Refinement models stay optional.

The main APK should not silently carry every model. Larger model assets are distributed separately so users can choose what to download.

Current model package

The release model ZIP is intended for users who want image refinement features. Keep models on GitHub Releases so the app and website can link to the same trusted artifact.
