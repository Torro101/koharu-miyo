Release notes

What MIYO focuses on

A reader built around ownership and speed.

01

Offline-first reading

Downloaded chapters are treated as first-class local content, reducing unnecessary network use when reading what is already on disk.

02

Smart download workflow

Download management is tuned for background work, cancellation paths, stuck-state visibility, and robust local storage handling.

03

Native performance helpers

C++ integrations support faster startup-adjacent work, image probing, archive handling, and refinement paths where native code is useful.

04

Source and plugin separation

Source-specific logic stays separate from the core app through parser infrastructure and guarded external plugin imports.

05

Image refinement tools

Optional refinement work can improve downloaded chapters while preserving safety checks against corrupted output.

06

Open development

MIYO is a rebranded continuation of Usagi, derived from Kotatsu, with visible credit and GPLv3 source availability.

Architecture

Simple surface, layered internals.

The app keeps reading, storage, parsing, downloads, refinement, and release update concerns separated so each layer can be improved without turning the reader into a single fragile workflow.

This website is only for MIYO project information and downloads. It does not host manga, source catalogs, or content mirrored from third-party providers.

  1. Readerpages, cache, progress, offline fallback
  2. Downloadsqueues, workers, notifications, storage validation
  3. Sourcesparser boundaries, auth, captcha handoff, plugin guards
  4. Nativeimage probes, archive work, performance helpers
